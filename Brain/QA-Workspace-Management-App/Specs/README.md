# Specs

Technical requirements, user stories, feature briefs. **What** should be built, not **how**.

## Conventions

- One file per feature or user story: `<feature-slug>.md`
- Use frontmatter: `status: draft | in-progress | shipped | abandoned`
- Link to related ADRs in a `## Decisions` section
- Link to the journal of the session where it was shipped

## Naming

`YYYY-MM-NN-<feature-slug>.md` or just `<feature-slug>.md` — pick one and stay consistent.
